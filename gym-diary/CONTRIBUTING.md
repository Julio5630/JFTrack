# Guia de Contribuição

Este documento descreve o processo de contribuição para o projeto Gym Diary.

## Começando

1. Faça um fork do repositório
2. Clone seu fork localmente
3. Crie uma branch para sua feature ou correção

## Desenvolvimento

### Setup Inicial

```bash
# Backend
cd backend
npm install

# Frontend
cd frontend
npm install
```

### Padrões de Código

- Use camelCase para nomes de variáveis e funções
- Use PascalCase para nomes de componentes e classes
- Adicione comentários para lógica complexa
- Mantenha funções pequenas e focadas

### Commits

Use mensagens de commit descritivas em português ou inglês:

- `feat: adiciona nova funcionalidade`
- `fix: corrige bug em...`
- `docs: atualiza documentação`
- `refactor: reorganiza código`
- `test: adiciona testes para...`

### Pull Requests

1. Descreva as mudanças realizadas
2. Reference as issues relacionadas
3. Teste as mudanças localmente
4. Adicione screenshots ou GIFs se aplicável

## Reportar Bugs

Ao reportar bugs, inclua:

- Descrição clara do problema
- Passos para reproduzir
- Comportamento esperado
- Comportamento observado
- Screenshots ou logs de erro

## Sugerir Features

Ao sugerir novas features:

- Descreva o caso de uso
- Explique como seria útil
- Mostre exemplos de implementação se possível

## Código de Conduta

Por favor, mantenha uma comunicação respeitosa e profissional em todas as interações.

---

Obrigado por contribuir para Gym Diary!
